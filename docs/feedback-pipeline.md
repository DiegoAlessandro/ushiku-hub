# ユーザー要望フィードバック・パイプライン

このドキュメントは、ユーザーからの改善要望を爆速で受け取り、OpenClawが自動的にサイトへ反映する仕組みを定義します。

## 🛠 仕組みの構成

1.  **入力**: サイトのフッターに「改善要望を出す」ボタンを設置（Googleフォームまたは専用API）。
2.  **トリガー**: 要望が届くと、DiscordまたはWebchat経由でOpenClawに通知が飛ぶ。
3.  **審査 (AI)**: OpenClawが要望の内容を読み、「悪質・スパム」でないか、「技術的に可能か」を判断。
4.  **自動実行**: OpenClawが自律的にコードを修正し、GitHubへPush → Vercelデプロイにより即座に反映。

## 🚀 実装タスク (Growth 100 連携)

### ステップ1: フィードバック導線の設置 (#35 拡張)
- [ ] 各ページのフッターに「⚡️爆速改善要望」ボタンを追加。
- [ ] 要望入力フォーム（内容＋メールアドレス）の作成。

### ステップ2: OpenClaw連携ハブの構築
- [ ] 要望が届いた際にOpenClawのセッションに「[User Feedback]」として自動投稿される仕組み。

### ステップ3: 自動審査・実装プロトコルの定義
- [ ] OpenClaw用プロンプト: 「届いた要望を分析し、悪質でなければ即座にコードを修正せよ」の設定。

## ⚠️ 審査基準 (AI Policy)
- **許可**: デザインの微調整、情報の誤り修正、新規カテゴリー要望、使い勝手の改善。
- **拒否**: 誹謗中傷、公序良俗に反する内容、システムの根幹を破壊するような要求。

---
*この仕組み自体をOpenClawが自律的に保守・改善し続けます。*
